# 不变量与硬约束

这些规则比局部实现细节更重要。任务如果要突破这些约束，需要先写清楚原因。

## 数据结构不变量

- 画布节点必须符合 `ComponentNode` 结构。
- `root_page` 必须始终存在，且作为根画布节点。
- 节点位置数据必须使用数值型 `x / y / width / height`。
- AI 生成结果进入正式画布前，必须先经过归一化和校验。

## 依赖边界不变量

- `src/` 不能直接 import `backend/src/`。
- 前端访问后端只能通过 `src/api/` 或浏览器网络调用。
- 后端不能依赖前端组件实现或浏览器专用 API。

## 编辑器运行时不变量

- 画布事实源是 `useComponentsStore`，不要在多个地方私自维护第二份主状态。
- 新增物料组件时，至少同步检查：
  - `stores/component-config.tsx`
  - 对应 `materials/` 实现
  - 属性编辑面板是否支持该组件关键 props
- 预览模式下不应保留编辑态的选中副作用。

## AI 生成链路不变量

- 同步接口和 SSE 接口都必须返回兼容的节点结构。
- `pendingComponents` 只能表示暂存生成结果，不能长期替代正式画布状态。
- `tool_call`、`tool_result`、`component_patch`、`done` 的事件顺序如果变更，前端消费逻辑也要同步更新。

## 文档与协作不变量

- 只要修改了结构边界、验证方式或关键用户流，就要同步更新 `docs/`。
- 非 trivial 任务不应只靠聊天历史，必须在 `plans/active/` 留下仓库内上下文。

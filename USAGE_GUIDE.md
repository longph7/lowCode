# 拖拽功能使用指南

## 🎯 功能概述

我已经为你的低代码编辑器实现了增强的拖拽功能，现在你可以享受以下特性：

### ✨ 主要功能

1. **精准插入预览** - 拖拽时显示蓝色插入线，精确指示放置位置
2. **智能位置计算** - 自动计算最佳插入位置（前面、后面、内部）
3. **视觉反馈** - 容器高亮、动画效果、跟随鼠标的预览
4. **性能优化** - 节流函数、状态管理、避免不必要的重渲染

## 🚀 如何测试

1. **启动项目**
   ```bash
   npm run dev
   ```

2. **打开浏览器** 访问 `http://localhost:5174`

3. **测试拖拽**
   - 在左侧材料面板中，你会看到组件库和一个测试区域
   - 拖拽任意组件（Button、Container、Text等）到测试区域
   - 观察蓝色插入线的显示效果
   - 尝试在不同位置放置组件

## 🎨 视觉效果

### 插入线效果
- **蓝色渐变线条** - 指示精确插入位置

## 📝 自定义配置

### 修改样式
编辑 `DragPreviewStyles.css` 文件：
```css
.insert-line {
    background: your-color; /* 修改插入线颜色 */
    height: your-height;    /* 修改插入线高度 */
}
```

### 调整阈值
在 `dragUtils.ts` 中修改：
```typescript
calculatePreciseInsertPosition(clientOffset, container, 15) // 默认10px
```

### 修改动画
在 CSS 文件中调整动画参数：
```css
animation: insertLinePulse 2s ease-in-out infinite; /* 修改动画时长 */
```

## 🐛 故障排除

### 常见问题

1. **插入线不显示**
   - 检查容器是否有 `data-component-id` 属性
   - 确认容器使用了 `useEnhancedMaterialDrops` Hook

2. **拖拽不生效**
   - 确认 React DnD 已正确配置
   - 检查 `accept` 数组是否包含正确的组件类型

3. **性能问题**
   - 检查是否有过多的重渲染
   - 考虑增加节流延迟时间

### 调试技巧

1. **开启控制台** - 查看拖拽相关的 console.log
2. **检查元素** - 使用开发者工具查看 DOM 结构
3. **状态监控** - 使用 Zustand DevTools 监控状态变化
